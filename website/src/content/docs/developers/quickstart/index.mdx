This guide will walk you through the basics of setting up and using Jux toolkit in your project. 

<Aside>
  For using Jux Toolkit with the Jux editor see the [working with Jux Editor](/developers/jux-editor) guide.
  This guide is for projects using any framework. 
  If you're using React, please refer to the [Vite/React guide](/developers/getting-started/vite).
</Aside>

<details>
<summary>What is Jux?</summary>
Jux is a type-safe, zero-runtime CSS-in-JS solution with full interpolation capabilities. 
It leverages PostCSS to generate optimized stylesheets during build time, combining the flexibility of CSS-in-JS with the performance of traditional CSS.
</details>

Let's dive in and get Jux set up in your project!

#### Installation

The fastest way to get started with Jux is to use the Jux CLI. The CLI will help you set up a new project and generate the necessary configuration files.

<Steps>
    1. ###### Install Jux CLI
        <Tabs>
            <TabItem label={'npm'}>
                <Code code={`npm install -D @juxio/cli`} title={'Terminal'} frame={'terminal'} lang={'shellscript'} />
            </TabItem>
            <TabItem label={'pnpm'}>
                <Code code={`pnpm add -D @juxio/cli`} title={'Terminal'} frame={'terminal'} lang={'shellscript'} />
            </TabItem>
            <TabItem label={'yarn'}>
                <Code code={`yarn add -D @juxio/cli`} title={'Terminal'} frame={'terminal'} lang={'shellscript'} />
            </TabItem>
        </Tabs>

    2. ###### Initialize a new project

        This will install the necessary dependencies, create a `jux.config.ts` and `postcss.config.js` files in your project root.

        <Code code={`npx jux init --postcss`} title={'Terminal'} frame={'terminal'} lang={'shellscript'} />
        
        <details>
        <summary>More information</summary>
        :::tip
        Use the `--skip-deps` flag to skip dependency installation if you're using an integrated monorepo with a centralized package.json (e.g. Nx). In this case, don't forget to manually add Jux to your root package.json: `npm install @juxio/postcss @juxio/css`
        :::

        ###### Under the hood
        The previous step created `jux.config.ts`, which specifies the paths to your source files. 
        Jux will scan and parse these files to generate the CSS.
        <Code code={juxConfig} lang="ts" title={'jux.config.ts'} mark={3} />
        <br/>

        ###### Generating a CSS file

        By running the Jux build process you can generate a CSS file for static usage. 
        This file will include all styles used in your project, along with tokens and theme configurations.
        <Code code={`npx jux generate css -o ./src/jux.css`} title={'terminal'} frame={'terminal'} />

        <Aside>For more information on the generated CSS file, see the [PostCSS](/developers/getting-started/postcss) guide.</Aside>
        </details>
        <br/>

    3. ##### Add the Jux layers to your main CSS file:
        <Code code={`@layer juxbase, juxtokens, juxutilities;`} lang="css" title={'main.css'} mark={1} />

    4. ##### Start using Jux in your project

        Include the main CSS file in your project and start using Jux to style your components.
        <Code code={mainTsx} lang="tsx" title={'index.tsx'} mark={1} />

</Steps>

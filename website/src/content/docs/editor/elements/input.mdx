### Technical Definition
The Input Element in Jux represents an interactive control for web-based forms, allowing users to input data. Currently, Jux supports a simple text input type, with more types planned for future releases.

### Anatony and Styling

The input element is a single layer:

- No other elements can be placed inside the input
- You can style both the text value and placeholder directly on the input itself
- To switch between styling the value and the placeholder, change the 'input mode' property in the Dynamic Design Panel

### Props and States

Input properties includes:
1. Interactive state:
    - Default
    - Hover
    - Active
2. Input mode (determines which content the rest of the DDP will affect):
    - Value
    - Placeholder
3. Input text (text string)
4. Placeholder text (text string)

Input properties can be extended based on your design needs, allowing for custom properties to enhance functionality and design variations. However, core properties like 'disabled' can't be removed to ensure proper web behavior.

### Special Mechanics
The input has two key limitations:

- No other element can be placed inside it
- Clicking anywhere in its area triggers the focus state, ready for typing

### Live Mode
In live mode, you can:

- Test and visualize all interactive states of the input
- See the placeholder text set in the source/instance of the input
- Type any value inside the input and observe the placeholder text disappear

Jux supports typography tokens as composite tokens that combine the following properties:

- Font family
- Font size
- Font weight
- Line height
- Letter spacing

<CaptionImage caption="Typography Token">
  <Image src={typographyToken} alt="Typography Token" />
</CaptionImage>

Each field within a typography token can be individually tokenized by referencing another token of the relevant type:

- Font family - references a font family token
- Font size - references a dimension token
- Font weight - references a font weight token <Badge text='Coming soon' />
- Line height - references a dimension token
- Letter spacing - references a dimension token

Typography tokens consist of the following fields:

- Name (e.g., "heading-1" or "body-text")
- Folder path (e.g., "typography/core/heading-1")
- Font family
- Font size
- Font weight
- Alias to another token - `optional`
- Optional: Line height - `optional`
- Optional: Letter spacing - `optional`
- Optional: Description - `optional`

<CaptionImage caption="Typography Props">
  <Image src={typographyProps} alt="Typography Props" />
</CaptionImage>

Typography tokens can only be used in the text module.

To tokenize any typography field, click on the 'token icon button' that appears on the right side of the field when you hover over it. When a typography field is tokenized, its five subfields are disabled. To enable these subfields again, the typography token must be detached

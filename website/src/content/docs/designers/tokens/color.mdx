Jux provides a flexible and precise approach to color tokens: Color tokens combine a HEX value (e.g., #141414) with an opacity percentage (1-100).

The combination can be expressed as a single RGBA value, maintaining its status as a simple token.

<CaptionImage caption="Color Token Card">
  <Image src={colorTokenCard} alt="Color Token Card" />
</CaptionImage>

The color circular thumbnail will show both the HEX value and the unified opacity result if the opacity value is less than 100%.

<CaptionImage caption="Transparent Color Example">
  <Image src={transparentColorExample} alt="Transparent Color Example" />
</CaptionImage>

Color tokens consist of the following fields:

- Name (e.g., blue-500 or primary-bg)
- Folder path (e.g., color/core/blue-500)
- HEX value
- Opacity value
- Alias to another token - `optional`
- Description - `optional`

<CaptionImage caption="Color Props">
  <Image src={colorProps} alt="Color Props" />
</CaptionImage>

Color tokens can be applied in the [Dynamic Design Panel (DDP)](/editor/dynamic-design-panel) within the following modules:

- Background color
- Text color
- Border color
- Shadow color
- Asset color

<CaptionImage caption="DDP Color Field">
  <Image src={ddpColorField} alt="DDP Color Field" />
</CaptionImage>

To tokenize any color field, click on the 'token icon button' that appears on the right side of the field when you hover over it.

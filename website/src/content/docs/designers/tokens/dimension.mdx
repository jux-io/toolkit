Jux supports dimension tokens consisting of a positive number and a unit of measure. Currently, Jux supports the following units: PX, %, EM, REM, VH, and VW.

<CaptionImage caption="Dimension Token">
  <Image src={dimensionToken} alt="Dimension Token"  />
</CaptionImage>

- Note: Dimension fields can accept entries like 'auto', which cannot be tokenized.

<Badge text='Coming soon' />: Dimension tokens can be referenced by other composite tokens, such as:

- Font size, line height, and letter spacing in Typography tokens
- Border width in Border tokens
- X, Y, spread, and blur values in Shadow tokens



Dimension tokens consist of the following fields:
- Name (e.g., "space_xs" or "padding_top_input")
- Folder path (e.g., "dimension/core/space_xs")
- Number value + unit
- Alias to another token - `optional`
- Description - `optional`

<CaptionImage caption="Dimension Props">
  <Image src={dimensionProps} alt="Dimension Props" width={563} />
</CaptionImage>

Dimension tokens can be applied in the Dynamic Design Panel (DDP) within the following modules:

- Size (width, height)
- Padding and margin
- Vertical and horizontal gaps (when display is 'flex')
- Border width
- Corner radius
- Shadow measurements

<CaptionImage caption="DDP Size Module">
  <Image src={ddpSizeModule} alt="DDP Size Module" width={563} />
</CaptionImage>

To tokenize any dimension field, click on the 'token icon button' that appears on the right side of the field when you hover over it.
